### Why This Works
- User (bixgomez) can read/write for development and git operations
- Apache (www-data) can read/write for WordPress updates and plugin installations
- Other users have read-only access (or no access for wp-config.php)

## Access Information

### From Mac Browser

### From Mac File System (SSHFS Mount)

**Issue:** macOS Finder's "Connect to Server" (Cmd+K) with `sftp://` URLs launches VLC instead of mounting the server, because VLC has hijacked the sftp:// protocol handler.

**Solution:** Use SSHFS to mount the server as a volume

**Mount Command:**
```bash
# Create mount point (one time)
mkdir -p ~/mnt/fezziwig

# Mount the server
sshfs bixgomez@192.168.64.5:/var/www/html ~/mnt/fezziwig
```

**Access:** After mounting, `/var/www/html` from the server is available at `~/mnt/fezziwig` on your Mac and appears in Finder as a regular volume.

**Unmount:**
```bash
umount ~/mnt/fezziwig
```
Or eject from Finder like any other volume.

**Optional Aliases:** Add to `~/.zshrc` or `~/.bashrc`:
```bash
alias mount-fezziwig='sshfs bixgomez@192.168.64.5:/var/www/html ~/mnt/fezziwig'
alias unmount-fezziwig='umount ~/mnt/fezziwig'
```

**Alternative:** Use Cyberduck (free SFTP client) to avoid the VLC protocol handler issue.

### Field Group 1: Song Fields
**File:** `group_5ba48b2235dbc.json`
**Title:** Song Fields
**Applied to:** Custom post type "song"
**Last Modified:** February 13, 2022

This field group manages individual song/track information with the following fields:

1. **Author(s)** (`author`)
   - Type: Text
   - Purpose: Store the songwriter/composer name(s)

   - Type: Repeater (table layout)

   - Type: Text

   - Type: Textarea
   - Purpose: Full Bandcamp player embed code for the track

5. **Bandcamp Album ID** (`bandcamp_album_id`)
   - Type: Text
   - Purpose: Bandcamp album identifier for API/embed usage

6. **Bandcamp Track ID** (`bandcamp_track_id`)
   - Type: Text
   - Purpose: Bandcamp track identifier for API/embed usage

7. **Soundcloud ID** (`soundcloud_id`)
   - Type: Text
   - Purpose: Soundcloud track identifier for embed integration

8. **Lyrics** (`lyrics`)
   - Type: WYSIWYG Editor
   - Purpose: Full song lyrics with rich text formatting
   - Media upload: Disabled

## Migration from ACF to Native WordPress - January 8, 2026

**Status:** Implementation complete, ready for testing

### Overview

Migrated all Advanced Custom Fields functionality to native WordPress meta boxes to reduce plugin dependencies and gain more control over the codebase. All field functionality has been preserved while removing the ACF plugin requirement.

- **Admin UI:** Classic meta boxes (familiar WordPress interface)

### Helper Functions (meta-helpers.php)

Provides ACF-compatible API for templates:

```php
mf_get_field($key, $post_id)           // Get any meta field
mf_get_personnel($post_id)             // Get personnel array from JSON
mf_has_personnel($post_id)             // Check if personnel exists
mf_get_album_songs($post_id)           // Get song IDs for album
mf_get_menu_item_field($key, $item)    // Get menu item meta
mf_get_menu_item_image($item)          // Get menu item image (ACF-compatible)
```

### Data Storage

#### Simple Fields
Stored as individual post meta entries:
- `author` - Text
- `credits` - Textarea
- `bandcamp_embed_code` - Textarea (allows HTML)
- `bandcamp_album_id` - Text
- `bandcamp_track_id` - Text
- `soundcloud_id` - Text
- `lyrics` - HTML content (sanitized with wp_kses_post)

#### Personnel Repeater
Stored as JSON string in single post meta key:
```json
[
  {
    "name": "John Doe",
    "contribution": "Guitar"
  },
  {
    "name": "Jane Smith",
    "contribution": "Vocals"
  }
]
```

#### Song Relationships
Stored as serialized PHP array in album post meta:
```php
array(1, 5, 12, 23)  // Song post IDs
```

#### Menu Item Fields
Stored as post meta on nav_menu_item posts:
- `_menu_item_class` - CSS class string
- `_menu_item_image` - Image URL
